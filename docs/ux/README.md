# ReOrg AI — UX Documentation

## Overview

This directory contains the UX specification for ReOrg AI, including navigation architecture, screen inventory, user flows, and component specifications.

## Contents

| File | Description |
|---|---|
| `routes.md` | Application route definitions |
| `flows.mmd` | User flow diagrams (Mermaid format) |
| `screens.md` | Screen inventory and descriptions |
| `figma-brief.md` | Figma design system brief |
| `ia_navigation.md` | Information architecture and navigation map |
| `data_intake_wizard.md` | Data upload wizard specification |
| `phase_workspace_template.md` | Reusable phase workspace layout |
| `outputs_report_generator.md` | Outputs and report generator specification |

## Design Principles

1. **Progressive disclosure**: Show complexity only when needed
2. **Evidence-first**: Every metric is clickable to reveal source evidence
3. **Status visibility**: Document and phase status always visible
4. **Guided workflow**: Wizards and gates guide users through the methodology
5. **Enterprise-grade**: Professional, information-dense but organized UI
